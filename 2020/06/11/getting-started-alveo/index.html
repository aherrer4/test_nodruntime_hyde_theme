Getting started with Xilinx Alveo U50 100Gbps designs

11 Jun 2020

In late 2018, Xilinx released a new series of production-ready boards under the name of Alveo family, featured with custom-built Virtex Ultrascale+ FPGAs. They are meant to accelerate Artificial Intelligence and Machine Learning tasks with ad-hoc “shields” that can be loaded on the device. To develop your applications, Xilinx released a platform called Vitis, which provides High-Level Synthesis tools to accelerate the development on these boards.

Here at Axbryd, we targetted the Alveo U50 to start to develop our 100Gbps applications. The choice was due to their relatively low-price and our expertise with the NetFPGA-SUME, our previous target platform. Unfortunately, since those devices are targeted as “Datacenter Acceleration Cards”, they do not come with any reference designs.

In this post, I will show you how you can set up a 100Gbps datapath to quickly integrate your HDL business logic and start developing on the Alveo U50 with plain Xilinx Vivado. No particular licenses for synthesis are needed. Although I haven’t tried on other boards of the same family, the process should be easily portable.

Xilinx has made the Alveo Family part of the Xilinx University Program (XUP). More info about donation requests in this Linkedin post.

Getting Started

Download and install the latest version of Vivado. This guide has been tested on Vivado 2019.2 and 2020.1.

Once the installation is finished, download the necessary files from here and extract it in your home directory.

The zip contains a modified version of the board files (the ones on the Xilinx site are not working as far of 13-Jul-2020) and the original XDC file.

To install the board files, if your Vivado’s installation path is /opt, just:

$ mv ~/au50 /opt/Xilinx/Vivado/20XY.Z/data/boards/board_files

Let’s create a new project called alveo_u50_datapath. Select RTL Project and skip the Add Sources and Add Constraints menu.

On the Default Part menu, let’s switch to Boards and select the Alveo U50 Datacenter Acceleration Card.

Beware that there is another similar board, the U50DD. This is a production sample, not meant for the market.

Block Design Initialization

Let’s create a new block design:

(Flow Navigator) IP Integrator --> Create Block Design

Let’s call it eth_subsys. Once the Block Design is open, head over to Board and, under the High Speed Transceivers, select the QSFP Connector. Right-click and select Connect Board Component. From the pop-up, change the component mode to qsfp_4x_322 and select the Ultrascale+ 100 Ethernet Subsystem. Click OK and w8 for the process to complete. Once finished, In the Diagram window, should appear a green banner with two clickable elements:

  • Run Block Automation
  • Run Connection Automation

Let’s click them sequentially and w8 for them to complete (if any pop-up appears, select All Automations and click OK).

At the end of those steps, you should have this Diagram:

Ethernet Subsystem customization and connection

Right-click on the Ultrascale+ 100 Ethernet Subsystem and then Customize Block. Under the General tab, change the following fields:

  • GT RefClk: 322.265625 MHz
  • User Interface: AXIS

Once finished, click OK and wait for the customization process to complete.

Let’s connect now the needed clocks and resets to the subsystem to work properly. The clk_out1 signal from the clock wizard is set at 100MHz. This clock need’s to be connected to:

Let’s now create a port for the clock signal which will feed our business logic. In the diagram, hit Ctrl-K and a pop-up appears. Name it clk_bl, set output as direction and Clock type. Repeat the step to create the reset_bl port, which is an Output and Reset type signal. Now we’re ready to connect the gt_txusrclk2 and the relative reset.

We need now to create a master reset signal. To do so, delete the conncetion from the clk_wiz reset signal and the usr_rx_rst signal. Add a sys_reset port, with Input direction of of type Reset and connect that to the reset port of the clk_wiz.

Now, head over the Sources tab, rigth-click on the eth_subsys and click on Generate HDL Wrapper.

Input and Output Queues

We’re now ready to generate the queues which will store our packets. Head over to the IP Catalog. Search for AXI4-Stream Data Fifo, double click on the element and choose Customize IP. Customize the IP with the following parameters and repeat the steps twice, for the input and output queue.

Component Name FIFO Depth TDATA Width (bytes) Enable TKEEP Enable TLAST
axis_input_fifo 4096 64 yes yes
axis_output_fifo 4096 64 yes yes

Use default options for the other parameters. When prompted with the Generate output products window, remember to select the Out of Context per IP option.

Synthesis and Implementation

Let’s now run the synthesis under Flow Navigator (this also may take a while). Once finished, let’s open the synthesized design. Let’s click on Edit Timing Constraints. Let’s create a new clock named clk_bl. Click on Source objects, select Nets in Find Names of Type: and filter by name with clk_bl. Move the ETH_SUBSYS_i/clk_bl to the rigth by clicking on the arrow. Click on Set, set the period to 3.103 ns and click on OK. Tick the Add clock checkbox for clk_bl and click on Apply in the All Constraints window.

Let’s now run the implementation and accept all the pop-ups that may appear (this will restart also the synthesi since we’ve changed the constraints).
